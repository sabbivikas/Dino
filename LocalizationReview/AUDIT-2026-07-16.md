# english-leak audit — 2026-07-16 (feature/localization, commit c429c11)

## what happened
owner reported the onboarding letter rendering in english in all four languages.
root cause CLASS: swiftui only auto-localizes `Text("literal")`. every string that
reaches the screen through a variable, ternary, interpolation-built string,
enum property, or system surface (notifications, siri, widgets, live activities,
share sheets, permission dialogs, a11y labels) bypassed the catalog.

## scope of the fix
- 903 flagged literals in 95 files → 7 classifier agents traced render paths → ~500 confirmed leaks
- 6 fixer agents + hand fixes wrapped every leak across 120+ files (stored/analytics
  values NEVER changed; display-only localization; english byte-identical)
- the known hole: Dino/Views/LetterView.swift typewriter letter (first launch) + onboarding
  welcome/navy quotes + option arrays + gratitude slips + testimonials + breath labels
- catalogs: Dino/Localizable.xcstrings 477→1308 · DinoLiveActivity/Localizable.xcstrings (NEW) 128
  · Dino/InfoPlist.xcstrings (NEW) 12 · DinoLiveActivity/InfoPlist.xcstrings (NEW) 1
  · Dino/AppShortcuts.xcstrings (NEW) 9 — 1458 keys, completeness sweep 0 gaps ×4 languages
- 828 newly translated keys per language in `{lang}-audit-additions.json` (this folder)
- server: expedition delivered-words now locale-aware end to end (deployed, dormant until ship)
- the alex-bug shape struck once more DURING the audit: dynamic-only keys (`.localized`
  runtime lookups) are invisible to the extractor and were initially never added to the
  catalog — caught by the korean runtime walk, 174 keys added, re-verified on screen
